Objectives

  • Use that Raspberry Pi (RPi) you have collecting dust!
  • Learn how to use SSH
  • Explore some basic networking

Note: The Raspberry Pi Foundation has done a great job documenting how to login remotely to your little princess...but for the sake of being thorough

SSH connection on your local network

  • With your RPi up and running it should have an IP address by this point. We can check the IP by logging in using a monitor or checking the router connecting the RPi to the internet
    • You should log into your router using admin priviledges (if you have never done this a quick google search should get you up to speed) and check for devices connected to the router. Some familiar faces shoud be present - one of which should be your RPi.
  • With your RPi address in hand you can locally connect to the server using SSH
    • SSH in a nutshell is a secure channel within an unsecure network (source: Wikipedia)
  • `ssh pi@0.0.0.0` 0.0.0.0 representing your local IP address
    • This will prompt you for your password the default password for your RPi is 'raspberry'
  • Once logged in this should bring you to the RPi terminal!

Alright you are some hot stuff logging in remotely to your RPi... Now how would you do that when you are not in the Local Area Network (LAN)?

SSH connection from anywhere

  • The RPi Foundation suggests you use this: weaved
    • I have nothing against weaved but one of our objectives is to learn some basic networking so this tutorial will do this the manual way
  • Bring up you admin page for your local router again and you should see an option for "Port Forwarding" select this option

Wait...what is "Port Forwarding"?

  • Your router's IP address is what every device on its network uses to access the internet.
  • Assuming you have multiple devices how does the router know which device is which?
    • Earlier in the tutorial we looked up our RPi IP address - this is how the router can distinguish between devices
  • Port forwarding allows requests from a client (in our case an SSH client) to be directed to the correct device on the correct port

OK...hopefully you kind of git the gist

  • Once on the Port Forwarding page we want to add a new item with a type TCP/UDP and enable global port forwarding
  • We should use port 22 in the port categoy of our new item
    • The RPi and other computers listen on port 22 for incoming SSH requests
  • Entering the local IP address we found earlier we have the last data point in order to begin port forwarding to our RPi

Now you should be able to connect to your RPi from anywhere!

  • Similar to how we logged in before (`ssh pi@0.0.0.0`) now we will use the routers IP address instead of RPi's IP address
    • You can now test this out from your LAN
